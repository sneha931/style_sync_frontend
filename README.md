# StyleSync 🎨

StyleSync is a web application that converts any website into an editable design system. It scrapes design tokens (colors, typography, spacing) and displays them inside an interactive, Figma-like dashboard.

## Tech Stack
- **Frontend**: React, Vite, TypeScript, TailwindCSS, Zustand
- **Backend**: Node.js, Express, Puppeteer, ts-node
- **Database**: PostgreSQL (via Supabase), Prisma
- **Styling Config**: Tailwind CSS with custom CSS variables generated by Zustand.

## Setup Instructions

### Environment Variables
For the **backend**, create a `.env` file in `stylesync/backend`:
```env
DATABASE_URL=postgresql://postgres:[PASSWORD]@db.[PROJECT-ID].supabase.co:5432/postgres
PORT=5000
NODE_ENV=development
```
(Replace with your exact Supabase PostgreSQL connection string)

### Database Setup (Supabase)
1. Initialize Prisma setup:
   ```bash
   cd backend
   npx prisma generate
   npx prisma db push
   ```
2. This creates Tables: `scraped_sites`, `design_tokens`, `locked_tokens`, `version_history`.

### Running the Backend
1. Navigate to the `backend` folder.
2. Install dependencies: `npm install`
3. Run the development server (runs with nodemon and ts-node):
   ```bash
   npm run dev
   ```
4. The server runs on `http://localhost:5000`.

### Running the Frontend
1. Navigate to the `frontend` folder.
2. Install dependencies: `npm install`
3. Start Vite dev server:
   ```bash
   npm run dev
   ```
4. The app runs via Vite, typically on `http://localhost:5173`. Make sure backend is running so that fetching tokens and scraping works.

### Deployment Instructions
1. **Frontend (Vercel)**: Connect Git repository to Vercel. Ensure Root Directory is `frontend` and output dir is `dist`.
2. **Backend (Render/Railway)**: Connect repository, set Root Directory to `backend`. Change build script to `npm install && npx prisma generate && npm run build` and start script to `npm start`.
3. **Database (Supabase)**: The database is permanently managed via the remote Postgres cluster url.
